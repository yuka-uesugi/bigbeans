## このアプリについて
- 名前: bigbeans（ビックビーンズ／バドミントンサークルの運営アプリ）
- 技術: Next.js（App Router）＋ TypeScript ＋ Firebase
- Firebaseプロジェクト名: `team-management-service`（本番。所有者は yuka.uesugi@all-in-one.jp）
- 似た名前の別アプリ `pkanri` が隣にある。間違えないよう注意。

## 確認・デプロイの標準手順
- ローカル確認: `cd /Users/uesugiyuka/Desktop/antigravty/bigbeans && npm run dev` → `http://localhost:3000`
- 本番反映: git push で Vercel が自動デプロイ（オーナーが「デプロイ」と言ったときのみ）
- Firestore/Storageのルール変更は git pushでは反映されない。**Firebaseコンソールの該当タブで手動公開**が必要（CLIは権限なし。下記参照）

## コマンド
- 開発サーバー: `npm run dev` ／ ビルド: `npm run build` ／ テスト: `npm test` ／ 型チェック: `npx tsc --noEmit`

## つまずきやすい運用ルール
- **Firebaseアカウントの食い違い**: ローカルのfirebase CLIは `yuka-uesugi@b-w-c.jp` で権限がない。本番プロジェクトへのルール反映は**必ずFirebaseコンソール**の「ルール」タブから行う。Firestore用とStorage用のルール画面を間違えないこと（URLで見分ける: `.../firestore/.../security/rules` と `.../storage/...rules`）。
- **会計の書込権限**: 「完全に管理者だけ」にはできない（参加費が予約確定時に自動記録されるため）。完全限定化にはCloud Functionsでの再設計が必要。
- **サポーター権限**: 代理編集の仕組みあり。Firestoreルールは対応済み。
- **予約解禁スケジュール**: 規約と実装場所はプロジェクトメモリー参照。
- 引き継ぎ再開の合言葉は「ビックビーンズ」（言われたら未了タスクから再開する）。
