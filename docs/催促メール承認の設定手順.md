# 集金「催促メール」承認機能 — 設定手順（管理者向け）

この機能を本番で動かすために、**Firebaseのコンソールで2つの確認・設定**が必要です。
プログラムの変更は完了しているので、ここを設定すれば動きます。

---

## 1. Firestoreルールに「承認待ち」の置き場所を追加する（必須）

幹事が催促を申請すると、`reminderRequests` という新しい置き場所にデータが入ります。
ここを読み書きできるように、ルールを1つ追加してください。

### 手順
1. Firebaseコンソール → 対象プロジェクト（team-management-service）を開く
2. 左メニュー「Firestore Database」→ 上部タブ「ルール」
3. 既存のルールの中（`match /databases/{database}/documents {` の内側）に、次を追加して「公開」

```
// 催促メールの承認待ちリクエスト
match /reminderRequests/{id} {
  // ログインしているメンバーなら、申請・閲覧・承認(削除)ができる
  allow read, write: if request.auth != null;
}
```

---

## 2. 承認依頼メールの宛先について（任意・確認のみ）

幹事が催促を申請したとき、「承認できる人（管理者・サポーター）」へお知らせメールが届きます。
この宛先は `userRoles`（権限の一覧）から自動で集めます。

- もし `userRoles` がログインメンバーから読める設定なら、そのままメールが届きます。
- もし読めない設定（管理者だけ等）の場合でも、**承認自体はアプリの「承認待ち」からできます**
  （お知らせメールだけが届かない状態になります）。

メールも確実に届けたい場合は、`userRoles` をログインメンバーが読めるようにしてください。

---

## 3. メール送信の合言葉（既存の設定）

催促メール・承認依頼メールは、既存の通知メールと同じ仕組み（クラブGmail）で送られます。
`GMAIL_USER` と `GMAIL_APP_PASSWORD` が本番（Vercel）に設定されていれば、実際に届きます。
未設定だと「メール送信の設定が未設定です」と表示されます。

---

## 動きのまとめ

- 集金リストは**全メンバーが作成可**。作った人が自動で「担当」になる
- 操作（納入チェック・催促・金額修正）は **担当者本人 or 管理者（管理者）** のみ
- 催促ボタン
  - 管理者が押す → その場で確認 → 催促メール送信
  - 幹事が押す → 管理者・サポーターへ承認依頼 → アプリの「承認待ち」で承認 → 催促メール送信
