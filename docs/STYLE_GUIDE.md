# チーム管理OS デザインガイドライン

本プロジェクトでは、大人のチーム運営にふさわしい「プレミアムで洗練された質感」を追求します。

## 絵文字の使用ルール（最重要・開発ルール）

「絵文字が多くて見にくい」状態を完全に排除し、プロフェッショナルな品質を維持するため、以下のルールを厳密に適用してください。

- **装飾用絵文字の完全禁止**: 
  - 見出し、ボタン、メニュー項目、リストの頭に「なんとなく」付ける装飾目的の絵文字（💰, 📸, 👥, 📝 等）は**一切禁止**です。
  - テキストのみで十分に伝わるよう、フォントの太さ (`font-bold`, `font-black`)、ウェイト、トラッキング（字間）、および適切な余白を設定してください。

- **機能的アイコンへの代替**:
  - 情報を視覚的に分類・指示する必要がある場合は、絵文字ではなく、SVGアイコン（Lucide React等）、カラーバッジ (`bg-ag-lime-500` 等の背景色)、あるいはシンプルな記号（`・`, `>` 等）を使用してください。
  - 例外的にシステム通知やステータスで絵文字を使用する場合も、1画面につき最小限（1〜2個）に留め、視覚的なノイズにならないよう配慮してください。

- **プレミアムな質感への昇華**:
  - 絵文字で表現していた「ジャンル」は、洗練された英語のラベル（`FINANCE`, `MEMBER`, `ALBUM` 等）を小さく添える、あるいはアイコンコンポーネントを単色で配置するなど、落ち着いた表現に置き換えてください。

## タイポグラフィと余白

- **情報の階層構造**: 絵文字による視覚的フックに頼らず、`text-xs` から `text-4xl` までのスケールと、`font-bold` / `font-black` の重みを使い分けて情報の優先順位を明確に表現します。
- **ゆとりある余白（Presbyopia-Friendly）**: 老眼対策を含め、コンテンツが密集しすぎないよう `p-8` や `gap-6` 以上のゆとりを基本とし、余白による「高級感」と「読みやすさ」を演出します。
- **一貫したデザイン言語**: ボタンやカードには `rounded-2xl` や `rounded-3xl` を多用し、柔らかくも洗練された、信頼感のある印象を与えます。

---

このガイドラインは、今後のすべての機能追加、UI 修正において適用される標準ルールです。
