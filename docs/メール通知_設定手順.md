# メール通知（第2段階）設定手順

作成日: 2026-06-09 ／ 更新日: 2026-06-18
対象: bigbeans 通知機能 第2段階（メール通知）

メール通知のプログラムは完成しています。
**会社のセキュリティ設定（サービスアカウント鍵のダウンロード禁止）に合わせて、鍵ファイルを一切使わない方式に変更しました。**
そのため、用意するのは **Gmailの「アプリパスワード」1つだけ** です。

---

## 用意するもの（1つだけ）

| 何 | どこで作る | 何に使う |
|---|---|---|
| Gmailの「アプリパスワード」（16文字） | 専用Gmailの設定画面 | メールを送るための合言葉 |

※ 以前必要だった「FirebaseのサービスアカウントJSON」は **不要になりました**。
　 本人確認はGoogleの公開鍵で、名簿の読み取りは投稿者本人の権限で行うため、秘密の鍵を持ちません（その分、より安全です）。

---

## ① Gmailのアプリパスワードを作る

送信に使う専用アカウント **`bigbeans.tsuduki@gmail.com`** で行います。

1. そのGmailアカウントでGoogleにログインする
2. ブラウザで `https://myaccount.google.com/security` を開く
3. **「2段階認証プロセス」をオンにする**（まだの場合）。電話番号でSMSを受け取る設定でOK
   - ※ 2段階認証をオンにしないと、次の「アプリパスワード」が作れません
4. ブラウザで `https://myaccount.google.com/apppasswords` を開く
5. 「アプリ名」に **bigbeans** と入力して「作成」
6. **16文字のパスワード**（例: `abcd efgh ijkl mnop`）が表示される
   - この16文字を**メモ**してください（スペースは消してOK）
   - この画面を閉じると二度と表示されないので注意

---

## ② 設定値を貼り付ける

設定する値は次の3つです（①以外はすでに入力済み）。

| Name（名前） | Value（値） |
|---|---|
| `GMAIL_USER` | `bigbeans.tsuduki@gmail.com` |
| `GMAIL_APP_PASSWORD` | ①の16文字（スペースは消してOK） |
| `NEXT_PUBLIC_SITE_URL` | 本番アプリのURL（例: `https://bigbeans.vercel.app`） |

### 手元（テスト環境）

`bigbeans` フォルダ内の **`.env.local`** に貼り付けます。
（`GMAIL_USER` と `NEXT_PUBLIC_SITE_URL` は設定済み。`GMAIL_APP_PASSWORD` に①を入れるだけ）

```
GMAIL_USER=bigbeans.tsuduki@gmail.com
GMAIL_APP_PASSWORD=ここに①の16文字
NEXT_PUBLIC_SITE_URL=https://bigbeans.vercel.app
```

### 本番（Vercel）

`https://vercel.com/` → bigbeans のプロジェクト → **Settings → Environment Variables** で
上の3つを追加（または更新）して Save。保存後に本番へ反映するため、一度デプロイし直します。

---

## テスト方法（ローカル）

1. `.env.local` に①を入れたら、開発サーバーを再起動（環境変数は再起動で読み込まれます）
   ```
   npm run dev
   ```
2. `http://localhost:3000` にログイン（メンバー以上）
3. **新しいお知らせ**（または予定・アンケート）を1件追加する
4. `bigbeans.tsuduki@gmail.com` と、自分のメールアドレスに**メールが届く**ことを確認
   - 届かないときは、ターミナルのエラー表示を確認（合言葉の貼り間違いが一番多い）

## 仕組みのメモ（技術者向け）

- 送信処理: `src/app/api/notify-email/route.ts`（Vercelのサーバー関数。Node.js）
- 本人確認・名簿読取: `src/lib/firebaseRest.ts`（`jose` で ID トークンを検証＋Firestore REST で名簿読取）
  - **サービスアカウント鍵（firebase-admin）は使わない**。会社のポリシーに準拠し、長期の秘密鍵をアプリに持たせない。
  - 本人確認: Googleの公開鍵（JWKS）で送信者の ID トークンを検証（メンバー以外＝無効なトークンは拒否）
  - 名簿読取: 送信者本人の ID トークンで Firestore REST API を呼び、`members` の最新設定を取得（Firestoreルール「メンバーは名簿閲覧可」に従う）
  - 名簿が読めない場合は、静的名簿（`memberList`）だけで送信を続行
- 呼び出し: `src/lib/notifications.ts` の `createBroadcast` 内から `/api/notify-email` をPOST
- 宛先: 名簿（`memberList`）＋ Firestore `members` の最新設定を突き合わせ、`notificationPrefs.practiceUpdates` が `email` または未設定の人だけに送る
- 宛先は **BCC** に入れて、メンバー同士にアドレスが見えないようにしている（個人情報保護）
- 受取ON/OFF: マイページ →「通知設定」→「予定・アンケート・お知らせの通知」で `メール / アプリ通知 / 受け取らない` を選べる
